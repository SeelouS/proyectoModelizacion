4  Bloque III: Clasificación Basada en Enlaces (Link-based).

Finalmente, explotaremos la topología de la red construida en el Bloque II para inferir información oculta del mercado.

  • Tarea: Utilizar la posición de un producto en el grafo para clasificarlo según diferentes métricas.
  • Reto: Si un producto no tiene categoría asignada, ¿podemos predecirla a partir de la categoría de sus vecinos en la red de usuarios (homofilia)? Usar la estructura del grafo para predecir si un producto pertenece a una clase “Elite” o para recomendar un enlace (Link Prediction), identificando qué productos suelen adquirirse de forma conjunta (co-compra).

4.1 Aclaración importante: ¿qué “categoría” predecimos?

El dataset con el que hemos estado trabajando no contiene una variable de categoría, precio o geográfica (ya se mencionó en el Bloque I). Por tanto, no existe una etiqueta de categoría “real” que ocultar y volver a predecir. En lugar de inventar una, haremos la predicción con etiquetas que sí podemos derivar honestamente de los datos y que juegan el papel de la “categoría desconocida”:

  1. Homofilia de la calidad: tomamos como etiqueta la calidad del producto (valoración alta/baja), y comprobamos si la posición en la red de un producto permite predecir su valoriación a partir de los vecinos. Por tanto, se intentará responder a la siguiente pregunta: “si no conociera la valoración de un producto, ¿podría adivinarla mirando a sus vecinos?”.
  2. Clasificación “Elite” desde la topología: predecimos si un producto es un best-seller usando solo descriptores estructurales del grafo.
  3. Link Prediction / co-compra: predecimos qué pares de productos deberían estar conectados (co-reseñados) y los proponemos como recomendaciones de co-compra.

4.2 Reconstrucción de las estructuras de los Bloques I y II.

Reconstruimos los resultados que validamos y estudiamos en los primeros dos bloques. Crearemos tanto la tabla de productos con las métricas usadas para aplicar PCA como el grafo de coparticipación con el que trabajamos en el .

library(data.table)
library(dplyr)
library(Matrix)
library(igraph)

set.seed(2004)   # misma semilla que en todo el proyecto

reviews <- fread("./Reviews.csv")

# --- Tabla de métricas por producto (como en el Bloque I, sobre TODOS los productos) ---
productos_full <- reviews |>
  mutate(size_texto = nchar(Text), size_summary = nchar(Summary)) |>
  group_by(ProductId) |>
  summarise(numero_reviews        = n(),
            rating_promedio       = mean(Score),
            size_summary_promedio = mean(size_summary),
            size_texto_promedio   = mean(size_texto),
            numero_usuarios       = n_distinct(UserId),
            .groups = "drop")

# --- Grafo de coparticipación (proyección producto–producto, umbral N = 2) ---
valores <- reviews |> select(UserId, ProductId) |> distinct()
pf <- factor(valores$ProductId)
uf <- factor(valores$UserId)

A <- sparseMatrix(i = as.integer(pf), j = as.integer(uf), x = 1,
                  dims = c(nlevels(pf), nlevels(uf)),
                  dimnames = list(levels(pf), levels(uf)))

M <- tcrossprod(A)                 # M[i,j] = nº de usuarios que reseñan i y j
M <- as(M, "generalMatrix")        # (forma no-simétrica; evita el aviso de 'dgCMatrix')
diag(M) <- 0

C <- M
C@x[C@x < 2] <- 0                  # umbral N = 2
C <- drop0(C)

g <- graph_from_adjacency_matrix(C, mode = "undirected",
                                 weighted = TRUE, diag = FALSE)
g <- delete_vertices(g, degree(g) == 0)   # limpieza de nodos aislados

cat("Grafo:", vcount(g), "nodos,", ecount(g), "aristas.\n")
Grafo: 12617 nodos, 101997 aristas.

Las tareas de clasificación e inferencia por enlaces solo tienen sentido sobre nodos conectados, por lo que trabajaremos sobre la componente gigante, que concentra la mayor parte de la señal estructural. Esto, además, es la forma eficiente de “manejar grafos de gran tamaño”: descartamos las ~1.900 componentes diminutas y nos quedamos con el núcleo del ecosistema.

comp <- components(g)
gcc  <- induced_subgraph(g, which(comp$membership == which.max(comp$csize)))
cat("Componente gigante:", vcount(gcc), "nodos,", ecount(gcc), "aristas",
    "(de", comp$no, "componentes en total).\n")
Componente gigante: 6855 nodos, 91832 aristas (de 1902 componentes en total).

4.3 Homofilia: ¿se puede predecir un atributo mirando a los vecinos?

La homofilia es el principio de que los nodos conectados tienden a parecerse. Si existe, la red por sí sola debería permitirnos imputar una etiqueta desconocida de un producto a partir de las de sus vecinos. Vamos a ponerlo a prueba con una etiqueta que no depende de la topología: la calidad del producto, definida como alta si su valoración media es \(\geq 4\) y baja en caso contrario.

rating <- productos_full |> select(ProductId, rating_promedio, numero_reviews)

vert <- tibble(ProductId = V(gcc)$name) |>
  left_join(rating, by = "ProductId")

vert$calidad <- factor(ifelse(vert$rating_promedio >= 4, "alta", "baja"))
V(gcc)$calidad <- as.character(vert$calidad)

table(vert$calidad)

4.3.1 Medida de homofilia: la asortatividad.

El coeficiente de asortatividad mide la correlación de una característica a lo largo de las aristas: vale \(+1\) con homofilia perfecta, \(0\) si la etiqueta se distribuye al azar y es negativo si los opuestos se atraen.

asort_nominal <- assortativity_nominal(gcc, as.integer(vert$calidad),
                                       directed = FALSE)
asort_numeric <- assortativity(gcc, vert$rating_promedio, directed = FALSE)

cat("Asortatividad (calidad alta/baja):", fmt(asort_nominal), "\n")
Asortatividad (calidad alta/baja): 0.133 
cat("Asortatividad (valoración continua):", fmt(asort_numeric), "\n")
Asortatividad (valoración continua): 0.194

Ambos valores son positivos pero pequeños (en torno a 0.13–0.19). Es decir: existe homofilia de calidad, pero es débil. Dos productos co-reseñados por los mismos usuarios tienden a tener valoraciones algo parecidas, aunque la relación está lejos de ser determinista. Esto ya anticipa una conclusión importante: la red codifica algo sobre la calidad, pero mucho menos de lo que codifica sobre la popularidad (lo veremos en § sec-elite).

4.3.2 Clasificación colectiva: voto ponderado de los vecinos.

Convirtamos la homofilia en un clasificador. Ocultamos la etiqueta del 30 % de los productos (conjunto de test) y los clasificamos por voto ponderado de sus vecinos del conjunto de entrenamiento (cada vecino vota por su clase con peso igual al nº de usuarios compartidos). Es el análogo sobre grafos de un \(k\)-NN.

set.seed(1)
n     <- vcount(gcc)
test  <- sample(n, round(0.30 * n))
train <- setdiff(seq_len(n), test)

lab <- vert$calidad
Aw  <- as_adjacency_matrix(gcc, attr = "weight", sparse = TRUE)  # pesos = usuarios compartidos

# Puntuación que cada clase recibe desde los vecinos de entrenamiento
idx_alta <- train[lab[train] == "alta"]
idx_baja <- train[lab[train] == "baja"]
sc_alta  <- as.numeric(Aw[, idx_alta] %*% rep(1, length(idx_alta)))
sc_baja  <- as.numeric(Aw[, idx_baja] %*% rep(1, length(idx_baja)))

clase_mayoritaria <- names(sort(table(lab[train]), decreasing = TRUE))[1]
pred <- ifelse(sc_alta == 0 & sc_baja == 0, clase_mayoritaria,
               ifelse(sc_alta >= sc_baja, "alta", "baja"))

acc      <- mean(pred[test] == as.character(lab[test]))
baseline <- max(prop.table(table(lab[test])))

# Matriz de confusión y exactitud balanceada (la clase está desbalanceada)
cm   <- table(real = lab[test],
              pred = factor(pred[test], levels = c("alta", "baja")))
sens <- cm["alta", "alta"] / sum(cm["alta", ])
spec <- cm["baja", "baja"] / sum(cm["baja", ])
bal  <- (sens + spec) / 2

Interpretación. El clasificador acierta un 77.3 %, por encima del baseline trivial de 73.9 % (que se obtendría diciendo siempre “alta”). Más informativa es la exactitud balanceada (0.70): al estar las clases desbalanceadas (~73 % son “alta”), lo relevante es que el modelo recupera la clase minoritaria “baja” con una especificidad de 0.56, muy por encima del azar. Conclusión: la estructura de co-reseña contiene señal real, aunque modesta, sobre la calidad de un producto —se puede imputar la etiqueta de un nodo mirando a sus vecinos, pero con un acierto limitado.

4.3.3 ¿Coinciden las comunidades con la calidad?

En el Bloque II detectamos comunidades con Louvain. Si las comunidades fueran “categorías de calidad”, cada una debería ser casi toda “alta” o casi toda “baja”. Veámoslo sobre la componente gigante.

set.seed(2004)
louv <- cluster_louvain(gcc, weights = E(gcc)$weight)

share_alta <- tapply(vert$calidad == "alta", membership(louv), mean)
grandes    <- share_alta[sizes(louv) >= 100]

cat("Comunidades:", length(louv), "| modularidad:", fmt(modularity(louv), 3), "\n")
Comunidades: 114 | modularidad: 0.788 
cat("Proporción de 'alta' por comunidad grande (rango):",
    fmt(min(grandes), 2), "–", fmt(max(grandes), 2),
    "| global:", fmt(mean(vert$calidad == "alta"), 2), "\n")
Proporción de 'alta' por comunidad grande (rango): 0.62 – 0.86 | global: 0.73

La modularidad es alta (0.79): hay una estructura de comunidades nítida. Pero la proporción de productos “alta” apenas varía entre comunidades y ronda siempre el valor global (~0.73). Es decir: las comunidades no organizan a los productos por calidad, sino por con quién se co-reseñan (afinidad de tipo de producto / “categoría” implícita). Esto encaja con la homofilia débil de calidad: la red agrupa productos por co-consumo, no por bondad.

4.4 Clasificación “Elite” desde la topología.

Cambiemos de etiqueta. Definimos un producto “Elite” como un best-seller: los del decil superior por número de reseñas. La pregunta es: ¿basta la posición en el grafo —sin mirar nunca el número de reseñas— para reconocerlos? Entrenamos una regresión logística usando solo descriptores estructurales.

feat <- tibble(
  pagerank   = page_rank(gcc, weights = E(gcc)$weight)$vector,
  strength   = strength(gcc, weights = E(gcc)$weight),
  betw       = betweenness(gcc, weights = 1 / E(gcc)$weight),  # peso = distancia = 1/similitud
  clustering = transitivity(gcc, type = "local", isolates = "zero"),
  grado      = degree(gcc)
)
TipDetalle técnico de betweenness

igraph interpreta los pesos de las aristas como distancias (coste de atravesarlas). Nuestro peso es una similitud (más usuarios compartidos ⇒ más cerca), así que pasamos weights = 1/weight para no invertir el significado. Es el mismo cuidado que conviene tener en el Bloque II.

elite <- factor(vert$numero_reviews >= quantile(vert$numero_reviews, 0.90))
datos <- bind_cols(feat, elite = elite)

set.seed(3)
te <- sample(nrow(datos), round(0.30 * nrow(datos)))
tr <- setdiff(seq_len(nrow(datos)), te)

modelo <- glm(elite ~ pagerank + strength + betw + clustering + grado,
              data = datos[tr, ], family = binomial)

prob <- predict(modelo, datos[te, ], type = "response")
y    <- datos$elite[te] == "TRUE"

# AUC mediante la equivalencia con el estadístico de Mann–Whitney
r   <- rank(prob); np <- sum(y); nn <- sum(!y)
auc_elite <- (sum(r[y]) - np * (np + 1) / 2) / (np * nn)

cat("AUC (test) del clasificador Elite:", fmt(auc_elite), "\n")
AUC (test) del clasificador Elite: 0.949 
cat("Prevalencia de Elite en test     :", fmt(mean(y), 3), "\n")
Prevalencia de Elite en test     : 0.105

Interpretación. El AUC es altísimo (0.95): la topología reconoce a los productos best-seller casi a la perfección. Esto contrasta de forma deliberada con la § sec-homofilia: la red predice muy bien la popularidad pero solo débilmente la calidad. La razón es estructural —un producto con muchas reseñas comparte usuarios con muchos otros y, por construcción, tiene grado/strength/PageRank altos—. Esta misma idea (centralidad \(\approx\) popularidad) será la clave de la validación cruzada con el PCA en el capítulo siguiente.

4.5 Link Prediction: predicción de la co-compra.

Última explotación de la topología: predecir aristas que aún no existen. Si dos productos comparten muchos vecinos en la red de co-reseña, es verosímil que un mismo usuario acabe reseñando (comprando) ambos: son candidatos a co-compra.

Validamos la idea de forma honesta: escondemos una muestra de aristas reales, puntuamos esos pares (positivos) frente a pares no conectados (negativos) usando solo la estructura restante, y medimos el AUC. Comparamos cuatro índices clásicos: vecinos comunes (CN), Jaccard, Adamic–Adar (AA) y preferential attachment (PA).

set.seed(7)
m_keep <- 3000                                   # muestra de aristas a ocultar
hold   <- sample(ecount(gcc), m_keep)
el     <- as_edgelist(gcc, names = FALSE)
gres   <- delete_edges(gcc, hold)                # grafo "residual" (sin las ocultas)

nb  <- adjacent_vertices(gres, V(gres))
deg <- degree(gres)

score_pairs <- function(pairs, method) {
  apply(pairs, 1, function(p) {
    a <- nb[[p[1]]]; b <- nb[[p[2]]]; inter <- intersect(a, b)
    if (method == "cn")  length(inter)
    else if (method == "jac") { u <- length(union(a, b)); if (u == 0) 0 else length(inter)/u }
    else if (method == "aa")  { w <- inter[deg[inter] > 1]; if (!length(w)) 0 else sum(1/log(deg[w])) }
    else deg[p[1]] * deg[p[2]]                   # preferential attachment
  })
}

pos <- el[hold, , drop = FALSE]                  # positivos = aristas ocultas
# Negativos: pares NO conectados en el grafo original, en igual número
clave_arista <- with(as.data.frame(el), paste(pmin(V1, V2), pmax(V1, V2)))
neg <- matrix(0L, nrow(pos), 2); i <- 0
while (i < nrow(pos)) {
  uv <- sample(vcount(gcc), 2); k <- paste(min(uv), max(uv))
  if (uv[1] != uv[2] && !(k %in% clave_arista)) { i <- i + 1; neg[i, ] <- uv }
}

auc_lp <- function(method) {
  sp <- score_pairs(pos, method); sn <- score_pairs(neg, method)
  r <- rank(c(sp, sn)); np <- length(sp); nn <- length(sn)
  (sum(r[seq_len(np)]) - np * (np + 1) / 2) / (np * nn)
}

aucs <- sapply(c(cn = "cn", jaccard = "jac", adamic_adar = "aa", pref_attach = "pa"), auc_lp)
round(aucs, 3)
         cn     jaccard adamic_adar pref_attach 
      0.973       0.971       0.975       0.914 
library(ggplot2)
tibble(metodo = names(aucs), auc = aucs) |>
  ggplot(aes(reorder(metodo, auc), auc, fill = auc)) +
  geom_col(width = 0.65, show.legend = FALSE) +
  geom_text(aes(label = fmt(auc, 3)), hjust = -0.15, size = 3.4) +
  coord_flip(ylim = c(0.5, 1.02)) +
  scale_fill_gradient(low = "#AFC6D9", high = "#2F5D8C") +
  labs(x = NULL, y = "AUC", title = "Link Prediction de co-compra") +
  theme_minimal()

Interpretación. Los tres índices basados en vecindario (CN, Jaccard y sobre todo Adamic–Adar, AUC \(\approx\) 0.98) predicen la co-reseña casi perfectamente, mientras que preferential attachment —que solo mira grados, no vecinos compartidos— queda por debajo. La lectura de negocio es clara: la co-compra es enormemente predecible a partir de la estructura local; conocer con quién comparte clientes un producto basta para anticipar con qué otros productos se emparejará. Adamic–Adar gana porque pondera a la baja a los usuarios “promiscuos” (que reseñan de todo) y da más valor a los vecinos compartidos selectivos.

4.5.1 Recomendaciones concretas de co-compra.

Apliquemos el índice de vecinos comunes a todos los pares aún no conectados (vía la multiplicación de matrices \(B \cdot B\), eficiente y dispersa) y mostremos los de mayor puntuación: son los pares de productos que el grafo sugiere que deberían comprarse juntos.

B  <- as_adjacency_matrix(gcc, sparse = TRUE)         # binaria
CN <- as(as(B %*% B, "generalMatrix"), "TsparseMatrix")  # vecinos comunes
nm <- V(gcc)$name

recs <- tibble(i = CN@i + 1L, j = CN@j + 1L, comunes = CN@x) |>
  filter(i < j, comunes > 0) |>
  mutate(clave = paste(pmin(i, j), pmax(i, j))) |>
  filter(!(clave %in% clave_arista)) |>                # solo pares NO conectados
  arrange(desc(comunes)) |>
  head(10) |>
  transmute(producto_A = nm[i], producto_B = nm[j], vecinos_comunes = comunes)

recs

Estos pares no comparten todavía \(\geq 2\) reseñadores directos, pero están rodeados de decenas de productos co-reseñados en común: son las recomendaciones de co-compra con mayor respaldo estructural.

4.6 Conclusión del Bloque III.

La topología de la red de coparticipación resulta ser un predictor muy desigual según qué queramos inferir:

  • Popularidad / prominencia comercial: se predice casi perfectamente desde la estructura (clasificador Elite, AUC \(\approx\) 0.95). La posición en el grafo es esencialmente una medida de popularidad.
  • Co-compra (Link Prediction): altísima capacidad predictiva (AA, AUC \(\approx\) 0.98). El vecindario local basta para recomendar qué productos se adquieren juntos.
  • Calidad / valoración: la red solo aporta señal débil (homofilia \(\approx\) 0.13; el voto de vecinos mejora el baseline, pero modestamente). La calidad es un atributo poco contagioso a través del co-consumo.

En conjunto, hemos usado los enlaces para clasificar productos (Elite), imputar un atributo por homofilia (calidad) y recomendar nuevos enlaces (co-compra), cubriendo las tres facetas del reto link-based. La idea de que centralidad ≈ popularidad es la bisagra que conecta este bloque con la validación cruzada frente al PCA.
