2  Bloque II: Análisis de Redes Sociales (SNA).

Una vez reducida la dimensionalidad, pasaréis a modelar el ecosistema de consumo como una red compleja bidimensional.

  • Tarea: Construir la red de usuarios en un grafo de coparticipación. Dos productos están conectados si han sido reseñados por los mismos N usuarios. Debéis justificar la elección de N basándoos en la densidad del grafo resultante.
  • Análisis: Utilizad métricas de centralidad: PageRank (para identificar productos “faro”), Betweenness (para identificar productos puente entre categorías), etc.
  • Detección de Comunidades: Mediante el algoritmo de Louvain o Walktrap, analizad si las comunidades coinciden con aspectos geográficos o de categorías de consumo u otras características.
  • Visualización: Mostrad las comunidades de productos más densas. Manejad eficientemente objetos de grafos de gran tamaño (limpieza de nodos aislados, simplificación de aristas).

2.1 Creación del dataset.

Para crear este dataset vamos a usar la librería Matrix. La intención es llegar a tener una matriz M, de tamaño \(P\times P\), donde P es el número de productos, y que la celda \((i, j)\) represente el número de usuarios que han reseñado tanto al producto i como al producto j.

Para esto es muy importante, quedarnos con los pares (UserId, ProductId), y que solo salgan una vez. Si un usuario ha reseñado más de una vez un mismo producto, solo contará como una fila.

#Paso 0: Cargamos el dataset, para ello usamos
#la librería data.table, como en el bloque anterior.
library(data.table)
library(dplyr)

# Usaremos esta semilla para todo el proyecto
set.seed(2004)

ds <- fread(file="./Reviews.csv")

#Paso 1: Nos quedamos solamente con los pares 
# (UserId, ProductId) únicos.

valores <- ds |>
  select(UserId, ProductId) |>
  unique()

# Paso 2: Pasamos esta información a una matriz dispersa.

library(Matrix)

productosFactores <- factor(valores$ProductId)
userFactores <- factor(valores$UserId)

A <- sparseMatrix(
  i = as.integer(productosFactores),
  j = as.integer(userFactores),
  x = 1,
  dims = c(nlevels(productosFactores), nlevels(userFactores)),
  dimnames = list(levels(productosFactores), levels(userFactores))
)

Ahora, creamos la matriz M que definimos anteriormente, y lo haremos a partir de la matriz que creamos en el bloque anterior (la matriz A).

M <- A %*% t(A)
M <- as(M, "dgCMatrix")
diag(M) <- 0          # quitamos auto-bucles

A continuación procederemos a elegir el umbral N más adecuado para nuestro grafo. Para ello, crearemos una función auxiliar densidad_para_N(N), que calculará la densidad, el número de componentes, y el tamaño de la componente más grande para un umbral N dado. Con esos datos, decidiremos cuál es el umbral más apropiado para nuestro grafo.

library(igraph)

densidad_para_N <- function(N) {
  C <- M
  C@x[C@x < N] <- 0 #pone a 0 los pesos por debajo de N
  C <- drop0(C)
  g <- graph_from_adjacency_matrix(C, 
                                   mode = "undirected",
                                   weighted = TRUE, 
                                   diag = FALSE)#no incluir diagonal
  tibble(
    N            = N,
    densidad     = edge_density(g),
    n_componentes = components(g)$no,
    size_componente_mas_grande = max(components(g)$csize)
  )
}

Probaremos con 10 umbrales distintos (desde \(N=1\) hasta \(N=10\)), y usaremos ggplot() para graficar la densidad resultante tras aplicar cada umbral. Para aplicar iterativamente la función densidad_para_N(N), usaremos la función map_dfr(v, f) de la librería purrr, que aplica (y guarda en un df) el resultado de aplicar la función f, a cada elemento del vector v.

Con la tabla y la gráfica podemos ver varias cosas:

  1. \(N=1\) es un umbral demasiado pequeño. El clúster (componente) más grande tiene un tamaño gigante. Casi 50000 productos de los \(\approx 72000\) que hay, están en esa componente.
  2. El codo se encuentra en \(N=2\), la caída de densidad de \(N=1\) a \(N=2\) es mucho más grande que la que hay de \(N=2\) a \(N=3\). Además, la componente más grande con \(N=2\) tiene menos de 7000 productos, reduciendo significativamente el tamaño de las componentes.

Por tanto, aunque \(N=3\) es también una buena opción, se escogerá \(N=2\) como umbral.

2.2 Creación y visualización del grafo.

Creamos nuestro grafo de coparticipación g con \(N=2\).

C <- M
C@x[C@x < 2] <- 0 #pone a 0 los pesos por debajo de N
C <- drop0(C)
g <- graph_from_adjacency_matrix(C, 
                                   mode = "undirected",
                                   weighted = TRUE, 
                                   diag = FALSE) #no incluir diagonal

Visualizaremos el grafo resultante para una exploración inicial. Usaremos ggraph, en combinación con graphlayouts, que incluye layouts pensados para grafos con muchos nodos y aristas. También identificaremos las comunidades que hay en el grafo usando el algoritmo de Louvain. Solo tendremos en cuenta componentes con más de 5 nodos para poder reducir un poco la complejidad del grafo y poder visualizarlo más fácilmente.

library(igraph)
library(ggraph)
library(graphlayouts)

g2 <- delete_edges(g, E(g)[weight < 2])
g2 <- delete_vertices(g2, degree(g2) == 0)

# comunidades + tamaño de componente como atributos de nodo
V(g2)$comunidad <- factor(membership(cluster_louvain(g2, weights = E(g2)$weight)))
V(g2)$tam       <- components(g2)$csize[components(g2)$membership]

lay_fun <- function(cmp) {
  if (vcount(cmp) > 100)
    layout_with_sparse_stress(cmp, pivots = min(500, vcount(cmp) - 1))
  else
    layout_with_stress(cmp)
}

set.seed(123)
xy <- layout_components(g2, layout = lay_fun)

ggraph(g2, layout = "manual", x = xy[, 1], y = xy[, 2]) +
  geom_edge_link0(edge_colour = "grey85", width = 0.1, alpha = 0.25) +
  geom_node_point(aes(colour = comunidad, size = tam),
                  alpha = 0.8, show.legend = FALSE) +
  scale_size(range = c(0.4, 2)) +
  theme_graph()

Estructuralmente, podemos apreciar que hay un gran clúster en el centro, que será la principal categoría de productos que se han vendido (Libros, tecnología, etc.), y después hay muchas componentes pequeñas.

Con respecto a las comunidades no podemos extraer mucha información, pues al haber tantas, los colores se “solapan”, es decir, hay muchos colores que son muy parecidos entre ellos. Sin embargo, hay una cosa en claro y es que la componente principal sí tiene una comunidad mayoritaria, cosa que también refuerza el concepto de categoría principal de producto.

2.3 Análisis de centralidad.

Veámos ahora qué productos son los más importantes tanto en el ranking proporcionado por PageRank (productos faro), tanto en el ranking proporcionado por Betweenness (productos puente).

Primero, convertiremos el grafo a un tbl_graph para simplifcar los cálculos y el tratamiento de los datos. Después, calcularemos tanto la métrica de PageRank como la de Betweenness.

library(tidygraph)

g <- g2 |>
  as_tbl_graph() |>
  activate(nodes) |>
  mutate(pagerank=centrality_pagerank(weights=weight,directed=FALSE),
         betw=centrality_betweenness(weights = weight, directed = FALSE),
         grado=centrality_degree(weights = weight))

2.3.1 Análisis de centralidad de PageRank.

Veamos los primeros 20 productos con mayor ranking en PageRank.

top_20 <- g |>
  activate(nodes) |>
  arrange(desc(pagerank)) |>
  as_tibble() |>
  select(-betw, -tam) |>
  head(20)

top_20

Podemos apreciar que los productos que más destacan son los 5 - 6 primeros, el resto mantienen un valor bastante parecido. Su valor no es muy elevado debido al gran tamaño de la red, que además tiene muchas componentes inconexas.

También podemos ver que los top-20 productos pertenecen a la misma comunidad (la 5). Luego es probable que sean productos similares, y que la gente suela comprar en conjunto.

2.3.2 Análisis de centralidad de Betweenness.

Veamos los primeros 20 productos con mayor ranking en Betweenness.

top_20 <- g |>
  activate(nodes) |>
  arrange(desc(betw)) |>
  as_tibble() |>
  select(-pagerank, -tam) |>
  head(20)

top_20

Aquí podemos ver que los productos que más destacan son también los 5 - 6 primeros; el resto mantienen un valor bastante más bajo que estos. Su valor es muy alto debido al gran número de componentes que tiene el grafo, siendo estos productos los pocos que conecten 2 o más componentes / comunidades entre ellas.

También podemos ver que los primeros 6 productos pertenecen a la misma comunidad que antes. Sin embargo, el que pertenezcan o no a esa comunidad enorme no es tan relevante aquí. Podemos observar que en el resto del top podemos encontrar productos que no son de esa categoría, aunque también es cierto que los productos más importantes (con respecto a esta métrica) sí son de la mega-categoría 5.

Por tanto, los productos más importantes (los productos puente) son:

2.3.3 Comprobación de los resultados.

Veamos en el grafo, usando el primer nodo de ejemplo (que resulta ser el mismo para las 2 métricas) quiénes son sus vecinos para comprobar que las métricas reflejan, en efecto, su importancia dentro de la red. Para ello crearemos un subgrafo con una egored.

g_i <- as.igraph(g)

# El ProductId líder es B002IEZJMA.

g_ego <- make_ego_graph(g_i, order = 1, nodes=V(g)[name=="B002IEZJMA"])[[1]]
V(g_ego)$ego <- V(g_ego)$name == "B002IEZJMA"




ggraph(g_ego, layout = "stress") +
  geom_edge_link(alpha = 0.08, colour = "grey70") +
  geom_node_point(aes(colour = comunidad, size = grado)) +
  geom_node_point(data = ~ subset(.x, ego), colour = "gold", size = 8, shape = 21, stroke = 1.2) +
  scale_colour_viridis_d(option = "turbo") +
  labs(colour = "Comunidad",
       title = "Ego-grafo de B002IEZJMA",
       subtitle = paste0(vcount(g_ego), " nodos · ", ecount(g_ego), " aristas"))

Aquí se demuestra claramente lo que ya exponían. Nuestro nodo (rodeado por un círculo amarillo), está conectado con muchísimos otros productos que tienen grado de centralidad elevados también (tamaño del nodo). Esto explica por qué tiene un valor de PageRank tan elevado.

También se puede apreciar que este nodo está conectado a muchas comunidades distintas (cada color representa una), cosa que explica por qué tiene un betweenness muy alto también. Esto se debe a que probablemente, estas comunidades distintas se conecten con pocos productos diferentes a los de su categoría, siendo nuestro nodo uno de ellos.

2.4 Conclusión.

En conclusión, podemos ver que hay varios nodos que destacan por su importancia. Algunos por sus conexiones con otros nodos importantes (PageRank), y otros por sus conexiones con otras “categorías” de productos; aunque es verdad que aquellos nodos que destacan especialmente en una métrica suelen destacar también en las demás, volviendo a reflejar la importancia que tienen estos nodos en la red.
